Глава 3.1. Введение в CSS

CSS — это язык, с помощью которого оформляют HTML-документы. Синтаксис CSS проще, чем синтаксис HTML: в нём нет сложных правил, связанных с вложенностью элементов. Да и вообще, в CSS почти нет вложенности, это достаточно «плоский» язык. Типичный стилевой файл состоит из списка CSS-правил, идущих друг за другом. Селекторы

Первая часть CSS-правила — селектор. Он определяет, к какому HTML-тегу или к какому множеству HTML-тегов применится набор свойств этого CSS-правила.

Селекторов великое множество, самые популярные из них расположены в примере. И чаще всего вы будете использовать три типа селекторов: по тегу, по классу и комбинированные селекторы, которые называют вложенными или контекстными.

Селектор по имени тега

Это самый простой тип селекторов. Такой селектор пишется так же, как имя тега, только без знаков меньше < и больше>, и выбирает все теги с подходящим именем. В примере выберутся все теги с именем р, то есть все абзацы.

Селектор по классу

Такой селектор начинается с точки, за которой идёт имя класса. Этот селектор выбирает все теги с подходящим значением атрибута class. В примере выберутся два элемента — p и span с классом title.

Можно комбинировать селекторы по тегу и по классу вот так: tag.class. Обратите внимание, что пробела в этой комбинации нет. Этот селектор выберет только подходящие теги с подходящим классом. В примере выберется только тег p с классом title, а тег span уже не выберется.

Селектор по идентификатору

Работает почти так же, как селектор по классу, только начинается с решётки, после которой пишется имя идентификатора, и выбирает все теги с подходящим значением атрибута id. В примере выберется тег h1 с идентификатором promo.

Селектор по идентификатору тоже можно комбинировать с селектором по тегу: tag#id. В этой комбинации тоже нет пробелов. В примере не выберется ни один тег, так как нет тега с подходящим id. Да, селекторы могут не выбирать ничего.

Селектор по идентификатору не рекомендуется использовать при стилизации. Сами по себе атрибуты id хороши и полезны, используйте их в своей HTML-разметке. Но в CSS коде от селектора по идентификатору нужно избавляться. Проблемы c этим селектором связаны с его излишней специфичностью, но о ней мы поговорим позже.

Вложенный (контекстный) селектор

Это комбинированный тип, когда вы берёте два селектора и соединяете их через пробел. Такие селекторы называются вложенными или контекстными и читаются справа налево. Селектор .promo a нужно читать так: «ссылки, которые лежат внутри тегов с классом promo». Причём эти ссылки могут располагаться и непосредственно в тегах с классом, и внутри тегов, вложенных в эти теги, на любой глубине. В примере выберется ссылка с текстом про магазин.

С помощью пробела можно комбинировать любые типы селекторов. В этом примере селектор выбирает все теги с классом warning внутри тегов с классом promo. Под это описание подходит только span с текстом про бритьё и стрижку.

В комбинации можно использовать и три, и четыре селектора, и намного больше. Чаще всего ограничиваются двойной вложенностью, изредка используют тройную, потому что, чем проще селекторы, тем проще ваш код.

Иногда в реальной разработке можно и нужно отходить от правила двойной вложенности, но всегда старайтесь работать либо с плоскими (то есть не вложенными) селекторами, либо с вложенными максимум до второго уровня.

Псевдоклассы и псевдоэлементы

Селекторы можно усиливать так называемыми псевдоклассами. Причем они бывают разными.

Есть псевдоклассы, которые помогают выбирать элементы с учётом их расположения относительно других элементов. Например, можно выбрать последний или первый элемент, каждый второй или каждый третий элемент и так далее.

Ещё есть псевдоклассы, которые добавляют возможность реагировать на состояния элементов. Например, было обычное поле ввода, потом на него навели мышкой и оно изменилось, затем в него поставили курсор, и это поле ещё как-то изменилось.

И ещё есть специальные псевдоклассы, которые добавляют возможность реагировать на состояние элементов форм. Например, можно выбрать все обязательные или, наоборот, необязательные поля формы.

Вот примеры использования псевдоклассов, в которых выбираются:

  • ссылки, находящиеся в фокусе;
  • каждый второй элемент списка;
  • поля обязательные для ввода;
  • отключенные кнопки.

В отличие от псевдоклассов, которые усиливают селекторы, псевдоэлементы порождают этакие виртуальные теги. Эти особые теги не существуют в HTML-коде страницы, а добавляются в документ из CSS-кода и подходят исключительно для оформления. Активно использовать псевдоэлементы мы начнём на этапе работы с мелкими декоративными элементами.

Чтобы ещё сильнее отделить псевдоклассы от псевдоэлементов, было решено изменить их синтаксис: псевдоэлементы пишутся с двумя двоеточиями ::, а псевдоклассы, как и раньше, с одним двоеточием :. Все современные браузеры понимают эти правила.

Если вам когда-нибудь понадобиться верстать для старых браузеров, можно просто дублировать селекторы с псевдоэлементами в старом синтаксисе. Причём, это можно делать автоматически с помощью соответствующих инструментов.

Чаще всего мы будем использовать псевдоэлементы ::before и ::after. Они хороши тем, что их можно добавить почти к каждому парному тегу. То есть у каждого парного тега на странице могут быть две дополнительные виртуальные сущности, которые можно использовать для стилизации. Это позволяет избавиться от тегов, которые нужны только для оформления и сделать разметку чище.

Чем отличаются ::before и ::after? Псевдоэлемент ::before добавляется внутрь тега до его содержимого, то есть первым элементом. Псевдоэлемент ::after добавляется внутрь тега после его содержимого, то есть последним элементом. Когда вы пишете такой вот селектор:

В нём описывается содержимое псевдоэлементов. Чаще всего в это свойство записывают просто пустую строку сontent: "";. Этого достаточно, чтобы псевдоэлемент отрисовался. Если не прописать это свойство, либо оставить его пустым, то псевдоэлемент не появится на странице.

Первую часть CSS-правила, а именно селекторы, мы разобрали. Синтаксис селекторов разнообразен, при желании можно накручивать очень сложные конструкции, но в львиной доле случаев вполне хватает простейших селекторов.

CSS-свойства

Вторая часть CSS-правила — это перечень свойств и их значений внутри фигурных скобок. Свойство и значение разделяются двоеточием, а пары свойств-значений разделяются точкой с запятой. Вот и весь синтаксис, он простейший. Сложность кроется в другом: CSS-свойств огромное количество. Но этого бояться не стоит, так как свойства можно легко разделить на независимые группы и изучать по-отдельности.

Спецификации CSS

В отличие от языка HTML, у которого есть одна спецификация, у языка CSS спецификаций много. Раньше спецификация CSS тоже была единой, но количество возможностей росло слишком быстро, спецификация пухла и развивать её как единый документ становилось всё сложнее. Поэтому её разделили. Сейчас существует большое количество отдельных спецификаций CSS, каждая из которых описывает какой-то аспект языка. Например, есть отдельные спецификации про анимации, трансформации, фильтры, флексы, гриды и так далее. В каждой такой спецификации описывается какое-то количество CSS-свойств, которые редко пересекаются со свойствами из других спецификаций.

При желании можно перечитать основные спецификации CSS, чтобы углубиться во все технические детали, но в начале пути мы вам не рекомендуем этого делать. Дело в том, что в спецификациях зачастую содержатся такие тонкости, которые вам пригодятся в одном случае из тысячи. Но через год-два, уже став профессиональными разработчиками, вы можете вернуться к вопросу глубокого изучения спецификаций CSS.

Каскадность

Теперь давайте поговорим о каскадности, ведь это тот самый механизм, который скрывается за первой С в аббревиатуре CSS.

При отрисовке страницы браузер должен определить итоговый вид каждого элемента. А для этого браузеру нужно собрать все CSS-свойства, которые к элементу относятся. Эти свойства могут «жить» в разных местах: в разных CSS-правилах внутри одного стилевого файла, в разных стилевых файлах, во встроенных в браузеры стилях по умолчанию и так далее.

И, как вы понимаете, CSS-правила могут пересекаться: на один и тот же элемент могут одновременно действовать несколько CSS-правил из разных источников. В примере на один и тот же абзац действуют CSS-правила по тегу и по классу. После того, как все правила для элемента собраны, браузер комбинирует все свойства из этих правил и применяет их к этому элементу. Ниже в примере к абзацу применятся жирность и цвет текста.

Описанный выше механизм, когда браузер для каждого элемента находит все связанные с ним CSS-правила и собирает для каждого элемента его собственный перечень CSS-свойств, и называется каскадностью. Учитывая то, что на современных страницах встречаются сотни тегов и CSS-правил, задачка получается достаточно трудоёмкой. Конечно, она оптимизирована в браузерах с помощью каких-то хитрых алгоритмов из области дискретной математики.

Каскадность позволяет разбивать код стилей на маленькие кусочки, а потом комбинировать их. Причем комбинирование делают намеренно, специально задавая тегу несколько классов. Как вы помните, тегу можно задать несколько классов, перечислив их через пробел внутри атрибута class. Простой пример — ссылки на социальные сети, которые часто делают в виде кнопок. У каждой такой ссылки обычно есть два класса: общий и частный. Общий класс задаёт всем кнопкам одинаковую геометрию, размеры и отступы, а частный класс задаёт разный фон. Такой подход позволяет сократить объём CSS-кода и сделать его понятнее. И всё благодаря каскадности.

В предыдущих примерах не было проблем с каскадностью, так как в разных CSS-правилах для тега были разные свойства. Проблемы начинаются, когда в разных CSS-правилах для тега встречаются одинаковые свойства с разными значениями. В этом случае возникает конфликт свойств. И браузеру нужно как-то решать, каким будет итоговое значение. В примере браузер выбирает, какой должен быть цвет фона: чёрный или синий?

Принцип каскадности с неизбежностью приводит к таким конфликтам, то есть для CSS это вещь достаточно обыденная. Разработчики языка это хорошо понимали, поэтому разработали правила разрешения этих конфликтов. Эти правила опираются на понятия приоритетности и специфичности. Мы сосредоточимся на понятии специфичности, так как именно с ней вы будете встречаться чаще всего.

В простейшем случае, когда у конфликтующих CSS-правил однотипные селекторы, выигрывает то правило, которое находится ниже в коде. В нашем примере как раз такой случай, и цвет фона в итоге будет синим. В более сложных случаях, когда у конфликтующих CSS-правил селекторы разного типа, в игру включается механизм определения специфичности.

Браузер определяет, у какого CSS-правила селектор специфичнее и это CSS-правило побеждает. Что значит специфичность? Специфичность определяет, какое количество элементов потенциально может выбрать селектор. Чем больше элементов может выбрать селектор, тем меньше специфичность. Чем меньше элементов может выбрать селектор, тем специфичность выше.

Например, селектор по идентификатору может выбрать всего один элемент, так как значения атрибута id должны быть уникальными в пределах страницы. Поэтому селектор по идентификатору очень специфичный и очень сильный. Селектор же по тегу, напротив, очень слабый и неспецифичный, потому что может выбрать огромное количество элементов, ведь для этого нужно всего лишь совпадение по тегу.

Есть и более близкие случаи. Селектор по классу и вложенный селектор по классу. Вложенный селектор по классу специфичнее, так как он может выбирать элементы не по всему документу, а только внутри другого элемента с подходящим классом.

Чтобы не сталкиваться с проблемами, связанными со специфичностью, старайтесь использовать максимально простые селекторы с небольшим уровнем вложенности. А также не используйте селектор по id — его слишком сложно переопределять.
